## Interpretation

- If TypeAware vs TypeAware-Consensus same_selection_rate is below 1.0000, the TypeAware-Consensus collapse is at least partially fixed.

- If objective_accuracy improves while leakage audit passes, the selectorfix is a valid improvement.
- If redundancy decreases but objective_accuracy does not improve, the selector behavior changed but did not translate into objective gains.
